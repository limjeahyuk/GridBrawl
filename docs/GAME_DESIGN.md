## ① 한 줄 요약

*이누야샤 데몬 토너먼트*처럼 **1:1 토너먼트 기반 카드 전투**를 핵심으로, 캐릭터 선택·성장과 짧은 플레이 세션을 결합한 그리드 액션 대전 게임.

- 장르: 턴제 / 카드 선택 / 전술 격투
- 한 턴에 **카드 3장**을 골라 순서대로 실행, 두 파이터가 동시에 진행
- HP가 0이 되면 승패 결정 → 다음 상대로 진행하는 토너먼트(건틀릿) 구조

## ② 참고작 분석 — Inuyasha: Demon Tournament

> 우리 게임의 "게임 방식"은 이 플래시 게임과 동일하게 가는 것이 목표입니다. 아래는 웹 리서치로 확인한 원작 룰입니다(출처는 문서 맨 아래).

### 개요
- 애니메이션 *이누야샤* 캐릭터로 싸우는 **턴제 1:1 전술 격투**. (단순화된 Tactics Arena Online + Street Fighter 느낌)
- 플레이어와 CPU가 **매 라운드 카드 3장을 선택**하고, 그 카드들이 **하나씩 차례로 공개되며** 행동이 펼쳐진다(양측 동시 진행).

### 전장(그리드)
- 2D 격자 전장에서 두 파이터가 마주 본다(스크린샷 기준 가로로 긴, 대략 3행 격자).
- 각 캐릭터는 격자 위 한 칸을 차지하고, **이동 카드**로 격자를 이동한다.

### 카드 구성
모든 카드에는 다음 정보가 표시된다: **이름 / 데미지(DM) / 에너지 비용(EN) / 3×3 사거리 그리드**.
- **3×3 그리드 표기**: 정중앙 칸 = 캐릭터 자신의 위치. 강조(빨강)된 칸 = 그 행동이 영향을 주는 범위.
- **이동 카드 (Movement)**: 카드 하단 화살표 방향으로 이동.
- **공격 카드 (Attack)**: 에너지를 소모, 상대가 사거리 칸 안에 있으면 데미지를 입힌다.
- **가드 카드 (Guard)**: 그 턴에 받는 피해를 **최대 15까지** 막는다.
- **에너지 회복 카드 (Energy)**: 에너지 바를 **15** 회복.

### 자원
- **에너지(EN)**: 공격/이동 등에 소모. 카드마다 비용이 다름. (라운드 사이 일부 회복)
- **HP**: 공격이 적중하면 카드 데미지만큼 차감. 0이 되면 패배.

### 진행/보상
- 적을 쓰러뜨리면 **사혼의 구슬 조각** 획득 + **미스터리 카드 5장 중 1장 선택**하여 덱에 추가(특수 공격 / 이동 강화 / 회복 등).
- 토너먼트(사다리) 형태로 점점 강한 상대와 대결.

### 캐릭터(원작)
이누야샤, 카고메, 셋쇼마루, 미로쿠, 산고, 키쿄, 나라쿠 등. 캐릭터마다 고유 기술/특수공격 보유.

> 📌 우리 게임은 원작 캐릭터를 **그대로 쓰지 않고** 오리지널 사이버 아레나 캐릭터(VOLT/TITAN/NOVA/CIPHER/AEGIS/EMBER)로 대체했습니다. 룰·시스템만 차용합니다(저작권 회피 + 오리지널리티).

## ③ 설계 명세 (의도) — 사용자가 정의한 룰

> 이것은 **목표로 하는 디자인**입니다. 현재 코드와 다른 부분이 있으며, 차이는 ⑤에 정리.

### 게임 흐름
1. 캐릭터 선택
2. 각 캐릭터는 **공용 카드 + 고유(직업) 카드**를 가진다.
3. 공용 카드: 이동 카드, 가드 카드, 에너지 회복 카드
4. 고유 카드: 공격 카드 (에너지를 써서 데미지. 카드마다 **범위 / 에너지 / 데미지**가 다름)
5. **매 턴 카드 3장 선택**
6. 상대와 함께 3장을 **순서대로** 실행
   - **카드 실행 순서: ① 이동 카드 → ② 수비(가드) 카드 → ③ 공격 카드**
7. HP가 다 닳으면 승리 또는 패배

### 시작 배치 (2D 격자)
```
+----+----+----+----+
| P1 |    |    | P2 |   ← 두 캐릭터가 격자 양끝에서 마주 보고 시작
|    |    |    |    |
+----+----+----+----+
```
(구현 확정: **3행 × 6열**. 두 파이터는 가운뎃줄 양 끝에서 마주 보고 시작.)

### 카드 명세

#### 1) 공용 카드 (Common)
| 카드 | 에너지 | 효과 | 쿨타임 |
|---|---|---|---|
| 이동 `>` `<` `^` `v` (1칸) | 0 | 해당 방향 1칸 이동 | 0턴 |
| 이동 `>>` `<<` (2칸) | 0 | 좌/우 2칸 이동 | 1턴 |
| 가드 | 10 | **50 데미지 방어** | 1턴 |
| 에너지 회복 | 0 | **에너지 50 회복** | 0턴 |

#### 2) 고유(직업) 카드 — 캐릭터마다 다름 (우선 한 캐릭터부터)
| 카드 | 범위 | 에너지 | 데미지 |
|---|---|---|---|
| 십자가 공격 | 캐릭터 상·하·좌·우 십자 범위 | 30 | 40 |
| 좌우 공격 | 캐릭터 좌·우 범위 | 40 | 50 |
| … (추가 예정) | | | |

> 핵심: **선택한 3장을 정해진 순서(이동→수비→공격)대로 실행하여 그에 맞는 행동을 하는 것**이 가장 중요한 메커니즘.

## ④ 현재 구현 상태 (코드 기준)

> ⚠️ 아래는 **실제로 코드에 들어가 있는 동작**입니다. ③의 의도와 다른 부분이 있으니 ⑤와 함께 보세요.

### 기술 스택 / 실행
- **React 19 + TypeScript + Vite** (SPA, 단일 캔버스형 무대).
- 개발: `npm run dev` · 빌드: `npm run build` · 타입검사: `npm run typecheck`
- 외부 게임 엔진 없음. 전투는 순수 TS 클래스(`CardBattle`)로 시뮬레이션하고 React로 렌더.

### 화면 흐름 (`src/App.tsx`)
`title → select(캐릭 선택) → bracket(대진표) → fight(전투) → result(결과)` 상태 머신.
- 토너먼트는 "건틀릿": 나머지 모든 파이터를 한 번씩 상대, 난이도 `easy,easy,normal,normal,hard`로 상승, 마지막이 최강. (`src/game/tournament.ts`)

### 전투 모델 (`src/battle/`) — **2D 격자** (방향 A로 구현 완료)
- **전장은 2D 격자**: `GRID_COLS = 6` × `GRID_ROWS = 3`. 셀 `{ col, row }`. col은 오른쪽으로, row는 아래로 증가.
- 시작 위치 `START_CELLS = [{0,1}, {5,1}]` — 가운뎃줄 양 끝에서 마주 봄. p0(플레이어)는 +col(오른쪽), p1(상대)는 −col(왼쪽)을 바라본다(`facing`).
- 자세(Stance) 개념은 **제거**됨 — 세로축은 이제 진짜 격자 이동(위/아래).
- 한 턴 = 카드 3장. **고른 슬롯 순서(1→2→3)대로** 해소하되, **한 슬롯 안에서** 나·상대 카드는 **타입 우선순위(이동 → 수비 → 공격)** 로 처리한다. 즉 "이동→수비→공격"은 전역 묶음이 아니라 *같은 슬롯 내 둘 중 누가 먼저 처리되는지* 정하는 규칙.

#### 카드 종류 (`CardKind`) — `src/battle/types.ts`
`move` | `attack` | `guard` | `energy`. 모든 카드에 `cooldown`(사용 후 잠기는 턴 수, 0/없음=재사용 가능).

#### 공용 카드 (`COMMON_CARDS`) — `src/battle/cards.ts`
| id | 이름 | kind | 효과 | 에너지 | 쿨타임 |
|---|---|---|---|---|---|
| `m-right` / `m-left` / `m-up` / `m-down` | 오른쪽/왼쪽/위/아래 | move | 해당 방향 1칸 (`>`/`<`/`^`/`v`) | 0 | **0** |
| `m-right2` / `m-left2` | 오른쪽/왼쪽 대시 | move | 좌·우 2칸 (`>>`/`<<`) | 0 | **1** |
| `c-guard` | 가드 | guard | 기력 **10**(`GUARD_COST`) 소모, 이번 턴 피해 최대 **50**(`GUARD_BLOCK`) 흡수 | 10 | **1** |
| `c-energy` | 원기 회복 | energy | 기력 **50**(`ENERGY_GAIN`) 회복 | 0 | **0** |
- 추가 페이싱 장치: **턴 시작 시 패시브 기력 회복 `ENERGY_REGEN = 30`** (양측, 최대치 cap). 명세에는 없던 디자인 보강 — 원기 카드가 쿨일 때 교착 방지.

#### 공격 카드 (캐릭터별, `src/data/roster.ts`)
공격 필드: `range`(타격 셀 오프셋 배열), `damage`, `energyCost`, `cooldown:0`.
- **`range`의 오프셋 `{ df, du }`**: `df`=앞쪽 칸 수(+는 상대 방향), `du`=위쪽 행 수(+는 위). 엔진이 `facing`으로 `df`의 좌우를 뒤집어 실제 셀로 변환(`targetsOf`).
- 적중 조건: 상대 셀이 변환된 타격 셀 집합에 포함될 때.
- 범위 헬퍼: `fwd(n)`(앞 n칸), `beam(a,b)`(같은 줄 직선 빔), `bar(df)`(앞 df칸의 위·중·아래 세로줄), `CROSS`(상하좌우 십자).
- 캐릭터(각 약/중/시그니처 3개): VOLT(장거리 빔), TITAN(앞줄 광역 거포), NOVA(최장 구체), CIPHER(**십자** 크로스 슬래시 = 명세 "십자가 공격"), AEGIS(**좌우** 벌워크 드라이브 = 명세 "좌우 공격"/탱커), EMBER(돌진 연격).

#### 캐릭터 패시브 (`Passive`, `src/data/roster.ts`)
각 캐릭터는 고유 패시브를 1개 가진다. 엔진이 정해진 시점에 자동 적용(`resolveTurn` 턴 시작 / `computeAttack` 공격 판정). 캐릭터 선택 화면 `detail__passive`에 설명 노출.

| 캐릭터 | 패시브 | 필드 | 적용 시점 |
|---|---|---|---|
| VOLT | 매 턴 기력 +10, 보호막 +10 | `turnEnergy:10, turnShield:10` | 턴 시작(전역 회복·실드 리셋 직후) |
| TITAN | 받는 공격 피해 **-10**(0 미만 불가) | `damageReduction:10` | 피격 데미지 계산(실드 흡수 후) |
| NOVA | 매 턴 기력 +10, 보호막 +10 | `turnEnergy:10, turnShield:10` | 턴 시작 |
| CIPHER | 공격 적중 시 **입힌 피해/5**(내림) 만큼 체력 회복 | `lifestealDiv:5` | 공격 적중(데미지 확정 후) |
| AEGIS | 매 턴 보호막 +20 | `turnShield:20` | 턴 시작 |
| EMBER | 공격이 적중하면 상대 보호막 **전부 제거** | `shieldBreak:true` | 공격 적중(실드 흡수 직전) |

- 적용 순서(공격 판정 한 건): `shieldBreak`로 상대 실드 0 → 실드 흡수 → `damageReduction` 차감 → 데미지 확정 → `lifestealDiv` 회복(공격자 HP, 최대치 cap).
- 패시브 보호막/기력은 가드·원기 카드 위에 **추가로** 쌓인다(같은 `s.shield`/`s.energy` 풀). 턴 시작 실드 리셋(`s.shield=[0,0]`) 직후 부여되므로 매 턴 갱신.
- `planAffordable`·UI 기력 예산은 플레이어 `turnEnergy`를 더해 계산(VOLT/NOVA가 더 비싼 플랜 가능).
- 회복은 `Step.heal`로 전달돼 전투 화면에서 초록 `+N`으로 표시(`.fighter__heal`).

#### 턴 해소 순서 (`CardBattle.resolveTurn`) — `src/battle/engine.ts`
턴 시작: 가드 실드 0으로 초기화 → 양측 패시브 기력 +30. 이어서 **슬롯 0→1→2 순서**로, 각 슬롯에서 그 슬롯의 두 카드(나·상대)를 **우선순위 `이동(0) < 수비(1) < 공격(2)`** 로 정렬해 낮은 쪽부터 처리:
- **이동(move)** — 해당 방향 이동(벽·상대 셀에 막히면 멈춤, 대시는 칸별로 진행).
- **수비(defense)** — `guard`(기력 10 지불, 실드 +50) / `energy`(기력 +50).
- **공격(attack)** — 정렬된 board(앞선 이동/수비 반영) 기준으로 판정. 사거리 밖=`whiff`, 기력 부족=`nofuel`, 적중 시 상대 실드부터 차감(전부 흡수되면 `blocked`).
- **한 슬롯에 양측 다 공격**이면(둘 다 우선순위 2) **동시 계산 후 동시 적용**(상호 트레이드·동시 KO 가능). 그 외엔 낮은 우선순위 카드가 먼저 → 다음 카드는 갱신된 board를 본다(예: 내가 먼저 이동하면 상대 공격을 회피, 상대가 먼저 가드하면 내 공격을 경감).
- 각 슬롯 처리 후 HP가 0 이하면 종료(둘 다 0이면 무승부 `winner=null`).

이 "슬롯 내 우선순위"가 명세의 의미다: *내 카드 3장은 고른 순서대로 실행*되고, 각 슬롯에서 *나와 상대 중 누가 먼저인지*만 이동→수비→공격으로 정해진다.

턴 종료: 쿨타임을 1씩 감소시킨 뒤, **이번 턴 사용한 cd≥1 카드를 풀 쿨타임으로 잠금** → `turn += 1`. `planAffordable`은 **선택한 슬롯 순서대로**(회복·가드·공격의 기력 증감을 순서대로) 지불 가능 여부를 검사.

#### AI (`src/battle/ai.ts`)
2D·쿨타임 인식 그리디 3장 플랜: ①사거리에 들면 공격(난이도별 `aggression`) → ②기력 낮으면 원기 → ③상대와 행·열 정렬(접근/대시) → ④가끔 가드 → ⑤폴백. 쿨타임/잠금을 존중하며 슬롯마다 자기 위치·기력을 시뮬레이션.

## ⑤ 설계 명세 ↔ 구현 정합 상태  ✅결정됨: **방향 A (2D 격자로 재작업)**

> 2026-06-15, 사용자 요청("위 다른 점들 다 바꿔주세요")으로 **코드를 설계 명세(2D 격자)에 맞추는 방향 A**를 채택·구현 완료. 원작과 명세가 모두 2D 격자 + 범위 마스크를 가리키므로 이 방향이 의도에 부합. 브라우저에서 한 턴 실행까지 검증됨.

## ⑥ 용어집
- **턴(Turn)**: 양측이 카드 3장을 골라 전역 페이즈로 해소하는 단위.
- **페이즈(Phase)**: 한 턴의 해소 단계 — `move`(이동) → `defense`(수비) → `attack`(공격).
- **셀(Cell)**: 격자 칸 `{ col, row }`. 파이터의 위치 단위.
- **오프셋(Offset)**: 공격 `range`의 한 칸 `{ df, du }` — `df`=앞쪽(+상대 방향), `du`=위쪽(+위). `facing`으로 좌우 반전되어 실제 셀로 변환.
- **facing**: 바라보는 방향. p0=+1(오른쪽), p1=−1(왼쪽).
- **실드(shield)**: 가드로 쌓는 턴 단위 피해 흡수 풀(가드 1장 = +50).
- **쿨타임(cooldown)**: 카드 사용 후 잠기는 턴 수. `state.cooldowns[p][id]`로 추적, 선택 시 0이어야 사용 가능.
- **스텝(Step)**: `resolveTurn`이 반환하는 개별 해소 결과(애니메이션 1컷). 행동자·결과·데미지·스냅샷 포함.
- **건틀릿(Gauntlet)**: 나머지 파이터를 차례로 모두 상대하는 토너먼트 구조.
- **시그니처(signature)**: 캐릭터의 대표 강공격(코드의 `signature: true`).
